# Призначення: максимально просте підключення

Вкладка `Призначення` працює так:

1. Google таблиця приймає відповіді з форми.
2. Apps Script віддає RichCore тільки необроблені рядки.
3. RichCore створює або оновлює картку в GitHub Projects.
4. RichCore фарбує рядок у Google таблиці як призначений, відхилений або знятий.

GitHub Projects:

- лідери: `UKRAINE-GTA-02` project `17`;
- заступники: `UKRAINE-GTA-02` project `9`;
- слідкуючі: `UKRAINE-GTA-02` project `7`.

## Найкоротше

Тепер нічого не треба міняти в коді Apps Script.

1. Відкрий Google таблицю з формою.
2. Відкрий `Розширення` -> `Apps Script`.
3. Встав весь код з [appointments_apps_script.gs](appointments_apps_script.gs).
4. Натисни `Deploy` -> `New deployment`.
5. Вибери `Web app`.
6. `Execute as`: `Me`.
7. `Who has access`: `Anyone with the link`.
8. Натисни `Deploy`.
9. Скопіюй `Web app URL`.
10. Встав цей URL у RichCore -> `Призначення` -> `Apps Script`.

Готово. Ніяких `API_TOKEN`, `SOURCES`, ID таблиць і назв листів вручну.

## Якщо таблиць дві

Якщо лідери / заступники в одній Google таблиці, а слідкуючі в іншій:

1. У першій таблиці встав і задеплой Apps Script.
2. У другій таблиці зроби те саме.
3. У RichCore встав обидва `Web app URL` в одне поле через кому:

```text
https://script.google.com/macros/s/AAA/exec, https://script.google.com/macros/s/BBB/exec
```

Якщо обидві форми лежать листами в одній Google таблиці, достатньо одного Apps Script URL. Скрипт сам перегляне всі листи.

## Як скрипт розуміє тип таблиці

Скрипт сам дивиться на заголовки колонок.

Лідери / заступники:

```text
NickName: | ID: | Посада: | Фракція: | Дата призначення: | Telegram: | Discord: | Посилання на форумний акаунт: | Електронна пошта: | 2FA (скрін):
```

Слідкуючі:

```text
NickName: | ID: | Дата призначення / погодження | Організація: | Telegram для зв'язку | Посилання на форумний акаунт: | Електронна пошта: | 2FA (двохфакторний захист акаунта):
```

Якщо є колонка `Посада`, RichCore визначає рядок як лідера або заступника. Якщо колонки `Посада` немає, але є `NickName` і `ID`, це слідкуючий.

## Кольори рядків

- білий рядок: нова заявка на призначення;
- зелений рядок: призначено, повторно не прийде;
- сірий рядок: відхилено, повторно не прийде;
- червоний рядок: заявка на зняття.

Після першого запуску Apps Script сам додасть і сховає службові колонки:

```text
RC_Status | RC_Action | RC_GitHub_Item_ID | RC_Updated_At | RC_Note
```

Їх руками чіпати не треба.

## Налаштування RichCore

1. Відкрий вкладку `Призначення`.
2. Apps Script URL і GitHub token уже стоять стандартно.
3. Перевір значення:

```text
OWNER: UKRAINE-GTA-02
Лідери: 17
Заступники: 9
Слідкуючі: 7
```

4. Натисни `Оновити список`.

## Робота щодня

Призначення:

1. Натисни `Оновити список`.
2. Обери людину.
3. Натисни `Копіювати тег` і додай її до Telegram групи.
4. Натисни `Копіювати оголошення`.
5. Натисни `Призначити`.
6. Якщо це лідер, натисни `Видати 12 ранг`.
7. Якщо це заступник, натисни `Видати 11 ранг`.

Зняття:

1. У Google таблиці познач рядок червоним.
2. У RichCore натисни `Оновити список`.
3. Обери людину.
4. Напиши причину.
5. Натисни `Копіювати оголошення`.
6. Натисни `Позначити знятим`.

Відхилення:

1. Обери заявку.
2. За потреби напиши нотатку.
3. Натисни `Відхилити`.

## Перевірка Apps Script

Після Deploy відкрий `Web app URL` у браузері. Має бути приблизно так:

```json
{"ok":true,"message":"RichCore Apps Script is ready"}
```

Якщо бачиш помилку доступу, перевір у Deploy:

- `Execute as`: `Me`;
- `Who has access`: `Anyone with the link`.
